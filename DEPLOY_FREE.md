# Публикация сайта

Цель: получить публичную ссылку на сайт и отдельную ссылку на админку без оплаты на старте.

## Что уже подготовлено

- Обычный сайт: `index.html`
- Админка: `/admin/`
- Редактируемые данные: `assets/content.json`
- Медиа для новых загрузок из админки: `assets/uploads/`
- Конфиг админки: `admin/config.yml`

Пользователи сайта не смогут ничего менять. Редактирование будет доступно только тем, у кого есть доступ к GitHub-репозиторию и админской ссылке.

## Вариант за 0 рублей

1. Создать аккаунт GitHub.
2. Создать репозиторий, например `absolut-site`.
3. Загрузить в репозиторий все файлы из этой папки: `outputs/absolut-site`.
4. Открыть `admin/config.yml` и заменить:

```yml
repo: REPLACE_WITH_GITHUB_LOGIN/absolut-site
```

на реальный путь, например:

```yml
repo: absolut-company/absolut-site
```

5. Создать аккаунт Cloudflare.
6. Открыть Cloudflare Pages, выбрать репозиторий GitHub и опубликовать сайт.
7. Для статического сайта build command не нужен, output directory можно оставить корнем проекта.
8. После публикации сайт будет доступен по ссылке вида `https://absolut-site.pages.dev`.
9. Админка будет доступна по ссылке `https://absolut-site.pages.dev/admin/`.

## Форма заявки

Форма может работать через подключенный сервис заявок. Если endpoint не указан, сайт открывает письмо на рабочую почту через почтовый клиент.

Для реальной отправки без открытия почтовой программы нужно подключить сервис заявок или серверный обработчик и вставить ссылку в поле `Endpoint формы` в админке. Сайт уже умеет отправлять форму POST-запросом на этот endpoint.

## Что можно менять в админке

- Телефон, почту, адрес, реквизиты
- Соцсети
- Карточки проектов
- Фото проектов
- Социальные объекты
- Полный реестр объектов
- Партнеров и логотипы
